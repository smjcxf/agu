> **生效日期：** 2026-06-17 起

> **修订原因：** 金股池容量膨胀（616只）、研报股不过期、评分区分度不足、上涨趋势条件过松、追板风险、止损一刀切

## 一、四大信号源（指标层）

### 1. 缠论买字（日K）

| 项目 | 说明 |
|------|------|
| **指标来源** | 缠论极点保留（逐bar循环版） |
| **核心逻辑** | 近3日K线出现局部低点（极点保留 == -1） |
| **计算方式** | 对250日K线逐bar判断极点，state传播确认 |
| **信号强度** | 近5日出现次数越多，信号越强 |

### 2. 金钻趋势

| 项目 | 说明 |
|------|------|
| **指标来源** | 金钻通道 + 回调买 |
| **核心逻辑** | 黄柱（金钻趋势 > HIGH）或 金钻起涨（DDX+量价） |
| **计算方式** | 通达信公式翻译：金钻趋势=EMA((H+L)/2,13)，起涨=DDX+量能+金叉 |
| **信号强度** | 黄柱为强信号，起涨为弱信号（但满足任一即算） |

### 3. 四量图机构

| 项目 | 说明 |
|------|------|
| **指标来源** | 四量图牛线/马线 |
| **核心逻辑** | 收盘价 > 牛线（机构持仓变红） |
| **计算方式** | MID9 = (3*C + L + O + H)/6，牛线1 = WMA20(MID9)，马线1 = MA(牛线1,6) |
| **信号强度** | 持续高于牛线天数越长，信号越强 |

### 4. 上涨趋势预警

| 项目 | 说明 |
|------|------|
| **指标来源** | 均线多头排列 + 金叉启动 + 主升浪 |
| **条件1** | MA5 > MA10 > MA20 > MA60，且所有均线均向上 |
| **条件2**（收紧） | MA60 > ref(MA60,1)，C > MA60，**MA5上穿MA20**（金叉），**V > ref(V,1) × 1.5**（量能确认），**MA5 > MA10**（新增） |
| **条件3**（收紧） | MA5 > MA10 > MA20 > MA60，C > MA5，**V > MAV5 × 1.2**（成交量高于5日均量20%），**MAV5 > MAV10 × 1.1**（5日均量高于10日均量10%） |
| **条件4** | 风控过滤：C > 0，V > 0，H > L（剔除停牌） |
| **综合判断** | (条件1 OR 条件2 OR 条件3) AND 条件4 |

> **2026-06-16 修订：** 条件2量能确认从1.3倍提高到1.5倍，新增MA5>MA10要求；条件3量价健康从V>MAV5收紧为V>MAV5×1.2，MAV5>MAV10收紧为MAV5>MAV10×1.1。

## 二、共振级别定义

| 级别 | 条件 | 含义 | 前端展示 |
|------|------|------|---------|
| **四喜临门（三足鼎立）** | 4个信号全部满足 | 最强信号，极其罕见 | 首页"四喜临门"卡片 |
| **三线共振** | 4个信号中满足 ≥3 个 | 高置信度买入信号 | 首页"三线共振"卡片 + 推荐列表 |
| **双线共振** | 4个信号中满足 2 个 | 观察级别，可跟踪 | 金股观测台按规则排序展示 |
| **单信号** | 仅满足 1 个 | 不纳入推荐 | **2026-06-16 起不再入池** |

## 三、金股池构建机制（修订版）

### 1. 扫描范围（不变）

| 板块 | 数量 | 数据来源 |
|------|------|---------|
| 创业板 | 100只 | 通达信直连 / 东方财富 |
| 科创板 | 100只 | 通达信直连 / 东方财富 |
| 主板 | 100只 | 通达信直连 / 东方财富 |
| 港股 | 50只 | 腾讯自选股 / efinance / 东方财富 |
| 投行研报关注池 | 30天内出现过的 | 观澜台抓取 |

### 2. 入池规则（2026-06-16 修订）

| 来源 | 修改前 | 修改后 | 说明 |
|------|--------|--------|------|
| **盘后全扫** | ≥1个信号就入池 | **≥2个信号才入池** | 单信号噪音过滤 |
| **盘中精监** | ≥2个信号入池 | ≥2个信号入池 | 不变 |
| **投行研报** | 无条件入池 | 无条件入池 | 来源独立，但过期统一处理 |

### 3. 保留规则（2026-06-16 修订）

| 来源 | 修改前 | 修改后 | 说明 |
|------|--------|--------|------|
| **信号来源股** | 保留10个交易日 | **保留20个交易日** | 观察周期延长 |
| **投行研报股** | 永久保留 | **保留20个交易日** | 取消永久保留，减少僵尸股 |

> **过期清理：** 所有股票统一按 `GOLD_POOL_DAYS = 20` 个交易日过期。即如果某只股票在入池后连续20个交易日都没有再次触发信号（含研报），则自动从金股池移除。

### 4. 预期效果

修改前：金股池总量约 **616只**（单信号占70%，研报僵尸股累积）  
修改后：金股池总量预计压缩到 **200-300只**（≥2个信号 + 20日过期）

## 四、综合评分公式（2026-06-16 全新升级）

### 公式

```python
score = signal_count * 3                    # 基础分（3/6/9/12）

# 连续共振天数奖励
days_in_resonance >= 3:  +2.0
days_in_resonance >= 2:  +1.0

# 动量适中加分
10% < pct20 < 30%:      +1.0
5% < pct20 < 50%:       +0.5

# 换手率活跃度加分
turnover_rate > 5%:      +1.0
turnover_rate > 3%:      +0.5

# 机构资金加分
fund_type == "机构":      +0.5
fund_type == "游资":      +0.3

# 研报覆盖加分
has_report:              +1.0

# 过热惩罚
pct20 > 50%:            -3.0
pct20 > 30%:            -1.0
pct20 < -20%:           -2.0

# RSI 风控
RSI > 70:               -1.0
RSI < 30:               +0.5

score = round(max(0, score), 1)
```

### 评分档位（修改后）

| 信号数 | 基础分 | 加分上限 | 扣分下限 | 实际范围 |
|--------|--------|---------|---------|---------|
| 1（不入池） | 3 | 约+5 | 约-5 | — |
| 2（入池） | 6 | 约+5 | 约-5 | 1.0 ~ 11.0 |
| 3（三线） | 9 | 约+5 | 约-5 | 4.0 ~ 14.0 |
| 4（四喜） | 12 | 约+5 | 约-5 | 7.0 ~ 17.0 |

> **修改前：** 评分只有 3/6/9/12 四档（RSI和换手率微调后仅 2.5/5.5/8.5/11.5 四档）  
> **修改后：** 多因子综合，评分分布从 0 到 17+，区分度大幅提升

### 前端展示

```javascript
// index.html / triple_resonance.html
function calcStockScore(s) {
  // 优先使用后端计算的综合评分
  if (s.signal_score !== undefined && s.signal_score !== null) {
    return Math.round(s.signal_score * 10) / 10;
  }
  // 兼容旧数据：按原公式计算
  return (s.signal_count * 3 + s.pct_chg * 0.5 + s.pct_chg_20d * 0.2).toFixed(1);
}
```

## 五、推荐生成规则（修订版）

### 1. 入推荐条件

- **三线共振**（signal_count ≥ 3）
- 按评分降序排列
- 同分则按连续共振天数降序
- 再同分则按涨跌幅降序

### 2. 涨停延迟机制（新增）

```python
涨停阈值 = 20.0 if 板块 in (创业板, 科创板) else 10.0
涨停延迟 = abs(当日涨跌幅) >= 涨停阈值
action = "WATCH" if 涨停延迟 else "BUY"
```

| 情况 | 前端展示 | 说明 |
|------|---------|------|
| 当日涨停 | `action: "WATCH"` | 标记为观察，建议延迟1天再推荐 |
| 未涨停 | `action: "BUY"` | 正常推荐 |

> **目的：** 避免追板风险，涨停当日买入往往次日低开。

### 3. 无三线时的处理（2026-06-16 修订）

| 修改前 | 修改后 | 说明 |
|--------|--------|------|
| 无三线时 recommend.json 为空 | **无三线时 recommend.json 为空（不变）** | 但金股观测台中双线共振股票仍可见，按规则排序 |

> 用户要求：无三线时不额外展示备选，金股观测台有按规则排序，可以看到双线共振。

## 六、止损/目标价（动态调整，2026-06-16 新增）

### 修改前

```python
stop_loss = close * 0.93  # 统一 -7%
target = close * 1.15     # 统一 +15%
```

### 修改后（基于板块 + 波动率）

| 板块 | 基础止损 | 基础目标 | 高波动调整（|pct20| > 30%） |
|------|---------|---------|---------------------------|
| 主板 | -7% | +15% | 止损放宽至 -10% |
| 创业板 / 科创板 | -10% | +20% | 止损放宽至 -15% |
| 港股 | -8% | +15% | 止损放宽至 -10% |

```python
# 高波动股票（近20日涨幅>30%或<-20%）
if abs(pct20) > 30:
    base_stop = min(base_stop + 0.03, 0.15)  # 最多放宽到-15%
```

> **目的：** 不同板块涨跌停限制不同，波动率不同，固定止损不合理。高波动股票需要更宽的止损空间。

## 七、三线共振历史数据（增强）

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `score` | number | 后端综合评分 |
| `days_in_resonance` | number | 连续共振天数 |

### 前端时间轴展示

```javascript
var currentList = stocks.map(function(s) {
  var score = calcStockScore(s);
  var daysText = s.days_in_resonance 
    ? ' <span style="color:#ff8a65;font-size:11px;">🔥连续' + s.days_in_resonance + '天</span>' 
    : '';
  return s.name + ' <span style="color:#999;font-size:11px;">(' + score + '分)</span>' + daysText;
}).join('、');
```

> 时间轴现在展示每只股票的评分和连续共振天数，帮助判断信号持续性。

## 八、数据流与执行流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 全盘扫描 (scanner.py --full)                       │
│  ├── 扫描350+只活跃股 + 研报股                              │
│  ├── 计算四大信号（缠论/金钻/四量图/上涨趋势）               │
│  ├── 计算综合评分（多因子）                                 │
│  ├── 涨停延迟标记（WATCH/BUY）                              │
│  └── 输出 scan_result.json                                  │
├─────────────────────────────────────────────────────────────┤
│  Step 2: 金股池更新 (gold_pool.py)                          │
│  ├── 读取旧金股池                                           │
│  ├── 添加新信号股（≥2个信号才入池）                         │
│  ├── 添加/更新投行研报股                                    │
│  ├── 清理过期股（20个交易日）                               │
│  └── 输出 gold_pool.json                                    │
├─────────────────────────────────────────────────────────────┤
│  Step 3: 推荐生成 (generate_recommend.py)                    │
│  ├── 提取三线共振（去重）                                    │
│  ├── 计算连续共振天数                                        │
│  ├── 按评分+连续天数排序                                     │
│  ├── 动态止损/目标价                                        │
│  └── 输出 recommend.json                                    │
├─────────────────────────────────────────────────────────────┤
│  Step 4: 历史数据 (generate_triple_resonance_history.py)   │
│  ├── 读取 gold_pool.json                                    │
│  ├── 生成每日三线共振快照                                    │
│  ├── 计算 score + days_in_resonance                         │
│  └── 输出 triple_resonance_history.json                     │
├─────────────────────────────────────────────────────────────┤
│  Step 5: 前端注入 (inject_macro.py + deploy)               │
│  ├── 宏观数据注入 index.html                                │
│  ├── 金股池数据注入 index.html                              │
│  └── 历史数据注入 triple_resonance.html                     │
└─────────────────────────────────────────────────────────────┘
```
