## 一、选股核心概念全貌

### 1. 四大信号源（指标层）

| 信号 | 指标来源 | 核心逻辑 | 权重 |
|------|---------|---------|------|
| **缠论买字** | 缠论极点保留（逐bar循环版） | 近3日K线出现局部低点（极点保留 == -1） | 等权 |
| **金钻趋势** | 金钻通道 + 回调买 | 黄柱（金钻趋势 > HIGH）或 金钻起涨（DDX+量价） | 等权 |
| **四量图机构** | 四量图牛线/马线 | 收盘价 > 牛线（机构持仓变红） | 等权 |
| **上涨趋势** | 均线多头排列 | 条件1(多头排列) / 条件2(金叉+量能) / 条件3(主升浪) 满足其一 | 等权 |

### 2. 共振级别定义

| 级别 | 条件 | 含义 |
|------|------|------|
| **四喜临门（三足鼎立）** | 4个信号全部满足 | 最强信号，极其罕见 |
| **三线共振** | 4个信号中满足 ≥3 个 | 高置信度买入信号 |
| **双线共振** | 4个信号中满足 2 个 | 观察级别，可跟踪 |
| **单信号** | 仅满足 1 个 | 纳入金股池观察，不推荐 |

### 3. 信号评分公式

```
评分 = signal_count × 3
if RSI > 70: 评分 -= 1  (过热扣分)
if 换手率 > 3%: 评分 += 0.5  (活跃加分)
最低分: 0
```

## 二、金股池构建机制

### 1. 扫描范围（成交量排序Top）

| 板块 | 数量 | 数据来源 |
|------|------|---------|
| 创业板 | 100只 | 通达信直连 / 东方财富 |
| 科创板 | 100只 | 通达信直连 / 东方财富 |
| 主板 | 100只 | 通达信直连 / 东方财富 |
| 港股 | 50只 | 腾讯自选股 / efinance / 东方财富 |
| 投行研报关注池 | 30天内出现过的 | 观澜台抓取 |

**总计扫描约 350+ 只活跃股 + 研报覆盖股**

### 2. 金股池入池规则

```
入池条件：
- 盘后全扫：只要有 ≥1 个信号就入池（门槛过低）
- 盘中精监：只要有 ≥2 个信号就入池
- 投行研报：无条件入池（不过期）

保留规则：
- 信号来源股保留 10 个交易日（自动过期）
- 投行研报来源股永久保留（不过期）
```

### 3. 推荐生成规则

```
推荐 = 三线共振（signal_count ≥ 3）
排序：连续共振天数降序 → 评分降序 → 涨跌幅降序
止损：-7%  目标：+15%
```

## 三、审计发现的问题

### 🔴 严重问题

#### 1. 金股池门槛过低，导致容量膨胀

| 指标 | 当前值 | 问题 |
|------|--------|------|
| 金股池总量 | 616只 | 远超有效跟踪上限 |
| 单信号入池 | 是 | 70%股票只有1个信号，噪声过大 |
| 研报股不过期 | 是 | 投行研报股永久累积 |

**影响：** 616只无法有效跟踪，淹没真正的三线共振信号。从截图看，168只"双重确认"（信号+研报）才是有效筛选，剩下448只是单信号噪音。

#### 2. 四喜临门（四信号全满）连续为0

从代码逻辑看：
- 缠论买字要求近3日出现局部低点
- 金钻黄柱要求金钻趋势 > HIGH（通道上穿）
- 机构变红要求收盘价 > 牛线
- 上涨趋势要求均线多头排列

这四个条件同时满足的概率极低，导致"四喜临门"栏位长期为空，浪费一个核心展示位。

#### 3. 上涨趋势条件过于宽松

```python
# 条件1 | 条件2 | 条件3 只要满足一个即可
选股信号 = (条件1 | 条件2 | 条件3) & 风控通过
```

- 条件1：MA5>MA10>MA20>MA60（标准多头排列）—— 严格
- 条件2：MA60上升 + 金叉突破 + 量能确认 —— 中等
- 条件3：多头排列 + 价格强势 + 量价健康 —— 中等（依赖条件1）

条件2和3与条件1有重叠，且条件2的"量能确认"（V > ref(V,1)*1.3）容易触发，导致"上涨趋势"信号含金量下降。

#### 4. 评分公式过于简单，缺乏区分度

```python
评分 = signal_count × 3
```

信号数只有1/2/3/4四个档位，评分只有3/6/9/12四档。加入RSI和换手率微调后，实际区分度有限。三线共振评分范围仅 8.5~12.5，导致截图中5只三线股评分从10.9到24.8跨度异常（怀疑有其他加分项未在代码中体现）。

### 🟡 中等问题

#### 5. 板块分布偏成长风格

| 板块 | 数量 | 占比 | 风险 |
|------|------|------|------|
| 创业板 | 121 | 19.6% | 高波动 |
| 科创板 | 122 | 19.8% | 高波动 |
| 主板 | 194 | 31.5% | 相对稳健 |
| 港股 | 68 | 11.0% | 汇率/政策风险 |

成长风格（科创+创业板）合计占40%，在大盘价值风格占优时，池子整体会跑输指数。

#### 6. 数据展示延迟

三线共振时间轴显示2只，但金股观测面板显示0只，说明前端数据刷新存在延迟或缓存问题。

#### 7. 缠论指标计算效率低

缠论极点保留采用逐bar循环（Python for循环），对每只股票的250日K线进行多轮嵌套循环，在全量扫描350+只股票时，计算开销巨大。建议改用 NumPy 向量化或 Cython 加速。

#### 8. 止损/目标价过于机械

```python
stop_loss = close * 0.93  # -7%
target = close * 1.15     # +15%
```

固定比例止损没有考虑：
- 个股波动率（ATR）差异
- 不同板块（主板/创业板）涨跌停限制不同
- 市场环境（牛/熊/震荡）

## 四、改进建议

### 立即执行（高优先级）

#### 1. 提高金股池入池门槛

```
建议修改：
- 盘后全扫入池门槛：signal_count ≥ 2（从1提高到2）
- 或：增加"最低评分 ≥ 6"作为门槛
```

预期效果：池子从616压缩到200-300只，提高跟踪效率。

#### 2. 给投行研报股加过期机制

```
建议修改：
- 研报股也保留30天（与关注池一致）
- 或：研报股只有在近10个交易日有信号时才保留
```

预期效果：减少僵尸股，释放池容量。

#### 3. 优化"四喜临门"展示逻辑

```
建议修改：
- 如果连续5日四喜为0，自动降级显示"最强三线共振"（评分最高的3只）
- 或：降低四喜门槛为"3.5线"（即3个核心信号+任意一个子条件）
```

### 中期优化（中优先级）

#### 4. 改进评分公式

```python
# 建议新评分公式
score = signal_count * 3  # 基础分
if 连续共振天数 >= 2: score += 2  # 持续性奖励
if 近20日涨幅 > 10% and 近20日涨幅 < 30%: score += 1  # 动量适中
if 近20日涨幅 > 50%: score -= 3  # 过热惩罚
if 成交额 > 10亿: score += 1  # 流动性奖励
if 研报覆盖: score += 1  # 机构背书
if fund_type == "机构": score += 0.5  # 机构资金加分
```

#### 5. 增加市场环境过滤

```
建议增加：
- 大盘RSI < 30时，暂停推荐（熊市保护）
- 涨停股票（pct_chg >= 10% or 20%）延迟1天推荐（避免追板）
- 高波动股票（ATR > 5%）提高止损比例到 -10%
```

#### 6. 缠论指标向量化优化

将 `calc_chanlun_signal` 中的逐bar循环改写为 NumPy 数组操作，预计可提升全量扫描速度 5-10 倍。

### 长期优化（低优先级）

#### 7. 增加信号回测权重

```
从 backtest 结果中学习：
- 哪个信号组合的胜率最高？
- 哪个板块的收益最好？
- 哪个资金类型的股票表现最优？
- 动态调整信号权重
```

#### 8. 增加多时间框架验证

当前仅使用日K，建议增加：
- 周线级别确认（大趋势方向）
- 60分钟级别入场时机（更精确买点）

## 五、审计结论

| 维度 | 评级 | 说明 |
|------|------|------|
| 信号逻辑 | ⭐⭐⭐⭐ | 四大信号互补，覆盖技术面+资金面+趋势面，逻辑合理 |
| 共振定义 | ⭐⭐⭐⭐ | 三线/四喜分级清晰，符合多因子选股思路 |
| 股池管理 | ⭐⭐ | 门槛过低，容量失控，研报股不过期是主要问题 |
| 评分体系 | ⭐⭐ | 过于简单，缺乏区分度和动态适应性 |
| 数据源 | ⭐⭐⭐⭐⭐ | 三重数据源+交叉验证，非常稳健 |
| 效率 | ⭐⭐ | 缠论逐循环计算效率低，扫描耗时长 |
| 风控 | ⭐⭐ | 固定止损缺乏弹性，无市场环境过滤 |

**总体评级：⭐⭐⭐（及格但需优化）**

**核心建议：** 先解决"金股池门槛过低"和"研报股不过期"两个问题，把池子从616压缩到200-300只，会立刻提升可用性。其他优化可逐步迭代。
